## 二、PDF 理论内容摘要

### 2.1 主课件：抵近任务轨道设计与可视化实践

#### 2.1.1 课程背景与抵近任务场景

主课件前半部分先说明课程目标，再把抵近任务放到任务背景中理解。核心信息是：

- 抵近轨道设计不是单一的“交会对接”问题，而是覆盖交会对接、近距离观测、在轨接触操作、小行星采样、空间目标抓捕等场景的统一设计问题。
- 课件把问题抽象为“严格终端位置、速度、姿态/指向约束下的近距离轨道设计”。
- 工程上常见任务模式包括经典直接交会、双共椭圆交会、共椭圆视线推力交会、V-bar 稳定点交会，以及稳定轨道与共椭圆组合交会。

#### 2.1.2 远程导引：Lambert 与 Lagrange 系数

主课件把抵近分为“远程导引 + 近距离交会”两段，其中远程导引以 Lambert 问题为主。核心关系是：

- 两点转移的 Lagrange 形式：
  - `r2 = f * r1 + g * v1`
  - `v2 = fdot * r1 + gdot * v1`
- 两端速度解算：
  - `v1 = (r2 - f * r1) / g`
  - `v2 = (gdot * r2 - r1) / g`
- 课件同时给出基于真近点角变化的 `f/g/fdot/gdot` 写法，以及基于通用变量的 Lambert 求解流程。
- 对于远程转移，机动脉冲定义为出发脉冲与到达脉冲：
  - `Delta v1 = v1 - v0`
  - `Delta v2 = vf - v2`
- 当前仓库中的 `MATLAB_Workspace/Final_Project/Remote_Guidance/lambert_universal_2d.m` 仅实现零圈（`0-rev`）Lambert；其中 `short/long` 仅表示单圈内的几何两支，不表示多圈 Lambert。

因此，本仓库中与 `f/g`、轨道递推、两点转移相关的内容，应优先对齐这一套 Lambert/Lagrange 写法。

#### 2.1.3 近距离交会：CW 与 TH

主课件将近距离相对运动统一为：

- 目标为圆轨道时使用 CW 方程。
- 目标为椭圆轨道时使用 TH 方程。

课件的建模逻辑是：

- 先从惯性系下追踪器与目标的绝对运动出发。
- 定义相对位置 `rho = rc - rt`。
- 再在“相对距离远小于目标轨道尺度”的假设下做线性化。

CW 模型的核心结论是：

- 适用前提：目标参考轨道近圆。
- 解析解存在，可直接写成状态转移矩阵分块形式。
- 本仓库使用的标准微分方程对应为：
  - `xddot - 2*n*ydot - 3*n^2*x = ax`
  - `yddot + 2*n*xdot = ay`
  - `zddot + n^2*z = az`

TH 模型的核心结论是：

- 适用前提：目标参考轨道椭圆。
- 一般没有像 CW 那样简洁的常系数解析状态转移矩阵。
- 课件给出了解析表达和从初值到积分常数的关系，用于椭圆目标轨道附近的相对运动建模。

#### 2.1.4 脉冲抵近任务轨道设计

主课件把多脉冲抵近整理成一套统一框架，关键量不是单个脉冲本身，而是：

- 总转移时间 `T`
- 转移段数/脉冲次数 `n`
- 中间脉冲点位置
- 相对距离 `rho(t)` 的趋近律

课件给出的标准边界条件是：

- `t = t0` 时，`rho(t0) = rho0 > 0`，`rhodot(t0) < 0`
- `t = T` 时，`rho(T) = 0`，`rhodot(T) < 0`

主课件总结了两类典型路径：

- 直线脉冲路径
  - 用目标反方向单位矢量定义空间路径方向。
  - 中间点按 `r_m = r_T + rho(t_m) * rho_hat` 形式生成。
- 抛物线脉冲路径
  - 用路径几何约束替代直线方向。
  - 适合需要控制视线角或满足附加路径约束的场景。

对 `rho(t)`，课件给出三类经典趋近律：

- 指数型
  - 相对速度与相对距离成比例。
  - 适合把抵近过程当作标准基准形式。
- 快速型
  - 以相对距离的负幂函数类形式构造。
  - 目标是在更短时间内完成逼近。
- 慢速型
  - 以相对距离的正幂函数类形式构造。
  - 目标是在更平缓的距离衰减中满足测控、光照、测量等约束。

多脉冲 `Delta V` 的课件框架是：

- 段内传播由 CW 状态转移矩阵给出。
- 由下一个路径点的位置约束反解本段脉冲后的速度。
- 总速度增量等于各段脉冲模长与末端速度修正模长之和。

#### 2.1.5 连续推力抵近与 LQR

主课件把连续推力抵近写成标准状态空间问题：

- `Xdot = A * X + B * u`
- 终端目标写成参考状态或目标状态。
- 再把跟踪误差写成：
  - `edot = A * e + B * u`

课件强调的核心思想是：

- 对抵近任务，通常更关心能量消耗而不是极快的过渡动态。
- 因此 LQR 权重应体现“昂贵控制”思路，即对控制输入施加较大代价。
- 课件给出圆轨道目标与椭圆轨道目标两种连续推力相对运动建模方式。

#### 2.1.6 主课件的补充知识块

主课件后半部分还有三个与仓库未来可扩展方向密切相关的补充块：

- 自然绕飞轨道
  - 包括目标为圆轨道时的平面自然绕飞，以及目标为椭圆轨道时的自然绕飞表达。
- 遍历巡游轨道
  - 关注多个目标的巡游、进入走廊约束、螺旋/巡游参数设计。
- 水滴悬停轨道
  - 关注相对椭圆中心漂移、水滴形态与悬停几何。

### 2.2 补充课件：二体问题基础知识

#### 2.2.1 开普勒定律与牛顿引力

补充课件从开普勒三定律出发，再回到牛顿定律和万有引力，形成“天体运动经验规律 -> 动力学机理 -> 二体简化模型”的完整链路。关键结论是：

- 椭圆律决定轨道几何基本形态。
- 面积律对应角动量守恒。
- 周期律把周期与半长轴关联起来。
- 牛顿第二定律与万有引力共同给出轨道动力学方程。

#### 2.2.2 从多体到二体的简化

补充课件先写一般引力叠加与外力项，再在卫星质量远小于中心天体质量、其他力可视为摄动时，得到二体简化。核心认识是：

- 两体之间的万有引力是主导项。
- 对近地卫星问题，可先以两体问题为基准，再视需要叠加摄动。
- 二体模型下的运动方程可以进一步推导出轨道守恒量与轨道几何方程。

#### 2.2.3 二体轨道几何与守恒量

补充课件把二体问题最常用的量整理为：

- 比角动量 `h = r x v`
- 面积速度守恒
- 轨道几何方程
- 圆锥曲线族：圆、椭圆、抛物线、双曲线

这一块与仓库里的椭圆轨道建模、近地点/远地点计算、轨道参数反解直接对应。

#### 2.2.4 坐标系与轨道根数

补充课件后半部分转向轨道描述方法，重点是：

- 常见地心/轨道坐标系的定义。
- 轨道根数的几何意义：
  - `a, e, i, Omega, omega, theta`
- 位置速度与轨道根数之间的互相转换。

这一部分是从几何和状态量回推轨道根数的基础，也是后续处理 PQW、LVLH、惯性系等转换的上游理论来源。

对于二维轨道上的 LVLH / 轨道坐标系构造，若已知目标的惯性系状态 `(r_t, v_t)`，更稳妥的做法是：

- 径向基向量取 `e_r = r_t / |r_t|`。
- 切向基向量由角动量方向确定，二维情形可写为 `e_theta = sign(h_z) * [-e_r_y; e_r_x]`，其中 `h_z = r_t_x * v_t_y - r_t_y * v_t_x`。
- 瞬时角速度应取 `theta_dot = h_z / |r_t|^2`。

因此，在数值传播或非严格圆轨道状态下，不应简单用 `v_t / |v_t|` 充当切向基，也不应把 `|v_t| / |r_t|` 直接当作 LVLH 角速度。

#### 2.2.5 时间函数与开普勒方程

补充课件最后给出椭圆轨道时间函数、偏近点角/平近点角以及开普勒方程。关键作用是：

- 把真近点角与时间关联起来。
- 给出椭圆轨道时间推进的标准参数化。
- 统一 `theta`、`E`、`M` 之间的关系。

这一部分直接支撑本仓库中椭圆轨道传播、`f/g` 计算、等待时间和飞行时间相关脚本。

### 2.3 常用符号

- `r`：位置矢量模长，或在上下文明确时表示相对位置向量。
- `v`：速度矢量模长，或在上下文明确时表示相对速度向量。
- `a`：半长轴。
- `e`：偏心率。
- `p`：半通径，`p = a * (1 - e^2)`。
- `rp`、`ra`：近地点距离、远地点距离。
- `T`：轨道周期或转移总时间，需结合上下文判断。
- `n`：圆轨道平均角速度，`n = sqrt(mu / r^3)`。
- `r0`、`v0`：初始状态。
- `rf`、`vf`：终端状态。
- `rho`：抵近阶段相对目标的距离标量。

## 七、综合大作业当前实现约定

### 7.1 远程导引段的二维共面简化

- 当前综合大作业第一阶段仅实现远程导引段，不实现近距离 CW 抵近设计。
- 目标圆轨道与跟踪器初始椭圆轨道被限定在同一轨道平面内，代码实现退化为惯性系 `x-y` 平面二维问题。
- 在该实现中，惯性系 `x` 轴沿跟踪器初始椭圆轨道近地点方向，目标初始相位角与跟踪器初始真近点角分别独立给定。
- 跟踪器状态直接由椭圆轨道时间函数 `M-E-theta` 推进得到，不通过 PQW 到惯性系的三维旋转变换。

### 7.2 捕获点定义

- 远程导引的 Lambert 终端点不取目标飞行器质心，而取目标 LVLH 坐标系中的固定捕获点 `rho_cap^L = [0, -5 km, 0]^T`。
- 实现时先由目标圆轨道状态构造 LVLH 基，再把该捕获点转换到惯性系，作为 Lambert 两点边值问题的终点位置。
- 终端速度同样按 LVLH 捕获点的运动学关系转换到惯性系，用于计算到达脉冲 `Delta v2`。

### 7.3 当前 Lambert 求解范围

- `MATLAB_Workspace/Final_Project/Remote_Guidance/lambert_universal_2d.m` 当前只求解零圈（`0-rev`）Lambert。
- 函数中的 `short` 与 `long` 仅表示零圈解中的几何 short-way / long-way 两支，不包含多圈 Lambert 的圈数枚举。
- 当前实现没有显式的 revolution-count 输入，也不会为同一组 `r1-r2-Delta t` 搜索多圈解族。
- 若后续任务需要多圈 Lambert，应单独扩展求解器的分支定义、`z` 搜索区间与主脚本结果记录结构。

### 7.4 远程导引段安全筛选判据

- Lambert 窗口的安全性不再按整条转移轨道的全局近地点筛选，而是按实际从 `r1` 飞到 `r2` 的那一段转移弧段最低高度筛选。
- 若转移弧段内径向速度从负值过渡到正值，则说明该弧段穿过近地点，弧段最小半径取该转移轨道近地点半径。
- 若转移弧段不穿过近地点，则弧段最小半径取出发点与到达点半径的较小值；未飞经的轨道部分不参与安全约束判定。
- 综合大作业当前安全约束写为 `r_min,arc >= Re + h_safe`。
